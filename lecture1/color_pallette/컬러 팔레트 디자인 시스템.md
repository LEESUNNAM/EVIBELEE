## CSS 변수 정의
```css
:root {
  /* Primary Colors */
  --color-primary: #7AAEC8;
  --color-primary-light: #A9CADB;
  --color-primary-dark: #3D5F73;

  /* Secondary Colors */
  --color-secondary: #5C8099;
  --color-accent: #C4D9EA;

  /* Background Colors */
  --color-bg-primary: #FFFFFF;
  --color-bg-secondary: #EEF5F9;

  /* Text Colors */
  --color-text-primary: #2B4A5E;
  --color-text-secondary: #3D5F73;
  --color-text-muted: #7AAEC8;

  /* Interactive Colors */
  --color-button-primary: #5C8099;
  --color-button-hover: #3D5F73;
  --color-link: #7AAEC8;
  --color-link-hover: #5C8099;
}
```

## 컬러 사용 가이드

| 변수 | Hex | 용도 | 사용 가이드 |
|---|---|---|---|
| `--color-primary` | `#7AAEC8` | 메인 브랜드 컬러 | 히어로 배경, 주요 섹션 배경, 브랜드 아이덴티티를 표현하는 영역에 사용. 넓은 면적에 배경으로 쓸 때 가장 효과적 |
| `--color-primary-light` | `#A9CADB` | 밝은 포인트 톤 | 카드 배경, hover 상태 배경, 섹션 내 강조 박스에 사용. primary보다 가볍고 부드러운 느낌 필요 시 적용 |
| `--color-primary-dark` | `#3D5F73` | 심층 다크 톤 | 푸터, 어두운 섹션 배경, 딥 오버레이에 사용. 포스터의 모서리 오버레이 효과를 웹에 구현할 때 적용 |
| `--color-secondary` | `#5C8099` | 보조 컬러 | 카드 테두리, 서브 섹션 배경, 부제목에 사용. primary와 같은 계열이므로 면적 비율을 primary의 절반 이하로 유지 |
| `--color-accent` | `#C4D9EA` | 구름 하이라이트 포인트 | 배지, 태그, 선택 상태 배경, 코드 블록 배경에 사용. 전체 팔레트에서 가장 밝은 블루 톤으로 산뜻한 강조에 적합 |
| `--color-bg-primary` | `#FFFFFF` | 기본 페이지 배경 | 모든 콘텐츠 페이지의 기본 배경. 텍스트 가독성 최우선 확보 |
| `--color-bg-secondary` | `#EEF5F9` | 섹션 구분 배경 | 교차 섹션 배경으로 사용해 콘텐츠 영역을 부드럽게 구분. 완전한 흰색과 블루 사이의 중간 톤 |
| `--color-text-primary` | `#2B4A5E` | 메인 텍스트 | 헤드라인(H1~H3), 핵심 본문에 사용. 흰 배경 위에서 충분한 명도 대비를 제공하는 다크 네이비 블루 |
| `--color-text-secondary` | `#3D5F73` | 보조 텍스트 | 본문(P), 설명 텍스트, 부제목에 사용. primary 텍스트보다 한 단계 밝아 계층 구조 표현 |
| `--color-text-muted` | `#7AAEC8` | 흐린 텍스트 | 날짜, 캡션, placeholder, 메타 정보에 사용. 14px 이하 소형 텍스트는 가독성 저하 주의 |
| `--color-button-primary` | `#5C8099` | 기본 CTA 버튼 | 주요 행동 유도 버튼 배경. 흰 텍스트와 조합해 충분한 대비 확보 |
| `--color-button-hover` | `#3D5F73` | 버튼 hover/active | 버튼 상태 변화 시 약 15% 어둡게 전환해 클릭 피드백 제공 |
| `--color-link` | `#7AAEC8` | 본문 링크 | 텍스트 속 링크가 primary 텍스트 색상과 구별되도록 primary 블루를 그대로 사용 |
| `--color-link-hover` | `#5C8099` | 링크 hover | hover 시 secondary 톤으로 진해지며 인터랙션 표현 |

## 반응형 고려사항

### 다크모드 적용
포스터 원본이 이미 어두운 블루 배경 + 흰 텍스트 구조이므로, 다크모드 전환이 이 팔레트의 가장 자연스러운 형태입니다.

```css
@media (prefers-color-scheme: dark) {
  :root {
    /* 배경 — 어두운 블루 계열로 반전 */
    --color-bg-primary: #2B4A5E;
    --color-bg-secondary: #3D5F73;

    /* 텍스트 — 흰색 계열로 반전 */
    --color-text-primary: #FFFFFF;
    --color-text-secondary: #C4D9EA;
    --color-text-muted: #7AAEC8;

    /* 버튼 — 밝은 톤으로 반전 */
    --color-button-primary: #7AAEC8;
    --color-button-hover: #A9CADB;

    /* 링크 — 밝은 accent으로 반전 */
    --color-link: #C4D9EA;
    --color-link-hover: #FFFFFF;
  }
}
```

### 화면 크기별 고려사항
- **모바일 (xs, ~600px)**: 히어로 섹션의 블루 그라데이션 배경은 단색 `--color-primary`로 단순화. 반투명 오버레이(backdrop-filter)는 성능 이슈가 있을 수 있으니 저사양 기기에서는 solid `--color-secondary`로 대체
- **태블릿 (sm~md, 600px~900px)**: 카드 그리드 2열 전환 시 `--color-bg-secondary`를 교차 섹션 배경으로 활용해 시각적 리듬 유지
- **데스크톱 (lg~, 1200px+)**: 포스터 원본처럼 넓은 히어로 배경에 `--color-primary` → `--color-primary-dark` 수직 그라데이션 적용 가능

### 접근성 주의사항
- `--color-text-muted` (`#7AAEC8`)는 흰 배경(`#FFFFFF`) 위에서 WCAG AA 기준(대비율 4.5:1)을 충족하지 않을 수 있으므로 **14px 이하 본문에는 사용 금지**
- `--color-button-primary` (`#5C8099`) 위의 흰 텍스트는 대비율 약 4.8:1로 AA 통과 — 버튼 최소 크기 44×44px 유지 권장
- 그라데이션 배경 위 텍스트는 배경의 가장 어두운 지점 기준으로 대비를 검증할 것
