# 기본 응답 설정

## 언어 설정
- 모든 답변은 한국어로 작성
- 정중하고 친근한 말투 사용
- 기술적 내용을 쉽게 설명

## 개발 원칙
- 코드 품질과 가독성 우선
- 사용자 요청사항에 정확히 응답
- 단계별 상세한 설명 제공

## GitHub 자동 업로드 규칙

- 이 저장소는 https://github.com/LEESUNNAM/EVIBELEE 와 연결되어 있다.
- **매 작업(사용자 요청 단위)이 끝날 때마다**, 변경된 파일을 위 저장소에 커밋 + 푸시한다.
- 업로드는 `/gh-cli` 스킬을 활용해서 진행한다 (인증은 `gh auth status`로 우선 확인).
- 절차:
  1. `git status`로 변경사항 확인
  2. 변경된 파일만 `git add` (전체 추가 `-A`/`.` 는 민감 파일 포함 여부를 반드시 먼저 확인한 뒤 사용)
  3. 작업 내용을 요약한 커밋 메시지로 `git commit`
  4. `git push` (최초 1회는 `git push -u origin main`)
- `.env`, 인증 토큰, `node_modules` 등 민감하거나 불필요한 파일은 절대 커밋하지 않는다 (`.gitignore` 참고).
- 푸시 완료 후 어떤 변경사항을 올렸는지 한 줄로 사용자에게 알린다.
